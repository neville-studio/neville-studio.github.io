本页面展示了由Neville Studio构建的OpenJDK版本。该构建以不更改OpenJDK官方发布的源代码为基本原则， 并且使用由OpenJDK推荐的构建要求进行构建。由于没有TCK测试套件，我们仅测试是否可以成功构建。

为什么要去构建Open JDK？ 主要是因为我想要一个可以在老旧系统上运行的Open JDK版本， 尤其是Windows, 我们将保持Open JDK 文档上要求的构建环境的系统要求进行构建。并且尽量不更新构建环境的系统

支持的运行平台:

  • Windows: Windows XP 或 Windows Server 2003(Open JDK 8, 部分API能够正常工作)
  • Windows: Windows Vista 或 Windows Server 2008(Open JDK 8, 所有API都能正常工作)
  • Windows: >=Windows 7 or Windows Server 2008 R2(>=Open JDK 8, 所有API都能正常工作，确保操作系统已完全更新)
  • Linux: GLIBC >= 2.12(Open JDK 8 或更高，部分功能需要fontConfig和x11等相关包)
  • Linux: 2.9 <= GLIBC < 2.12 (仅支持Open JDK 8，部分功能需要fontConfig和x11等相关包)
  • 其他: 不支持
  • 系统架构: 对于Open JDK 8: x86_64, x86; 对于>=OpenJDK 11, 仅x86_64

注意：请不要将此OpenJDK构建用于生产环境！你可以访问 Eclipse Adoptium的官方网站 去获取一个官方的稳定OpenJDK构建
